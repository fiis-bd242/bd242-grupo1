# 4.2. Módulo de Campaña Publicitaria

### 1. Caso de uso: Creación de Campaña Publicitaria

| **Realised User task**     | Crear y configurar una campaña publicitaria en el sistema de Yape. |
|----------------------------|--------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                         |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de gestión de campañas |
| **Flow of Events**          | 1. El administrador accede al módulo de campañas publicitarias.<br> 2. El sistema solicita la configuración inicial de la campaña (nombre, objetivo, duración).<br> 3. El administrador ingresa los detalles solicitados y selecciona los canales donde se lanzará la campaña.<br> 4. El sistema guarda los datos y muestra una vista previa de la campaña.<br> 5. El administrador revisa y aprueba la configuración.<br> 6. El sistema programa la campaña para su lanzamiento en los canales seleccionados. |
| **Exceptions**              | - **Datos incompletos**: Si los campos obligatorios no están completos, el sistema muestra un mensaje de error y no permite avanzar hasta que se completen.<br> - **Problemas de conexión**: Si el sistema no puede conectarse a los canales publicitarios seleccionados, muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - El usuario debe tener permisos de administrador de marketing.<br> - El sistema debe estar conectado a las plataformas de publicidad. |
| **Postcondition**           | La campaña publicitaria ha sido creada y programada para su lanzamiento. |
| **Includes Use Case**       | Revisión de campaña, Aprobación de campaña                         |
| **Used Services**           | - Servicio de gestión de campañas publicitarias <br> - Servicio de conexión con canales publicitarios |
| **Non-functional requirements** | - **Usabilidad**: El proceso de creación de campañas debe ser intuitivo para el usuario.<br> - **Seguridad**: Solo los usuarios autorizados pueden crear campañas.<br> - **Performance**: La campaña debe guardarse y programarse en menos de 2 segundos. |

---

### 2. Caso de uso: Monitoreo de Campaña

| **Realised User task**     | Monitorear el desempeño de una campaña publicitaria activa. |
|----------------------------|-------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                  |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de análisis de campañas |
| **Flow of Events**          | 1. El administrador accede al módulo de gestion de campañas publicitarias, especificamente en la parte de reportes.<br> 2. El sistema muestra un tablero con las métricas clave de la campaña (impresiones, clics, conversiones).<br> 3. El administrador puede filtrar los datos por fecha, canal o segmentación.<br> 4. El sistema actualiza el tablero en tiempo real. |
| **Exceptions**              | - **Problemas de conexión**: Si hay problemas para obtener datos de los canales, el sistema muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - La campaña debe estar activa y lanzada en los canales seleccionados.<br> - El sistema debe tener acceso a las métricas de los canales publicitarios. |
| **Postcondition**           | El administrador obtiene un informe actualizado del rendimiento de la campaña. |
| **Includes Use Case**       | Generación de informes                                       |
| **Used Services**           | - Servicio de análisis de campañas <br> - Conexión con los canales publicitarios |
| **Non-functional requirements** | - **Performance**: Las métricas deben actualizarse en tiempo real.<br> - **Disponibilidad**: El sistema debe estar disponible para monitorear la campaña 24/7. |

---

### 3. Caso de uso: Generación de Informes Post-Campaña

| **Realised User task**     | Generar un informe detallado del rendimiento de la campaña publicitaria una vez finalizada. |
|----------------------------|--------------------------------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                                                 |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de generación de informes                      |
| **Flow of Events**          | 1. El administrador selecciona una campaña finalizada para generar el informe.<br> 2. El sistema extrae los datos de rendimiento (impresiones, conversiones, CTR, etc.).<br> 3. El sistema genera el informe en formato PDF o Excel.<br> 4. El administrador descarga el informe o lo envía a otras áreas. |
| **Exceptions**              | - **Error en la extracción de datos**: Si el sistema no puede obtener datos completos de la campaña, muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - La campaña debe haber finalizado.<br> - Los datos deben estar almacenados y accesibles.  |
| **Postcondition**           | El informe detallado del rendimiento de la campaña ha sido generado. |
| **Includes Use Case**       | Exportación de informes                                                                     |
| **Used Services**           | - Servicio de generación de informes <br> - Servicio de almacenamiento de datos de campañas |
| **Non-functional requirements** | - **Usabilidad**: El proceso de generación de informes debe ser rápido y sencillo.<br> - **Performance**: El informe debe generarse en menos de 5 segundos. |

### 4. Caso de uso: Segmentación de Audiencia para Campaña

| **Realised User task**     | Definir y seleccionar el público objetivo para una campaña publicitaria. |
|----------------------------|---------------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                                 |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de segmentación de audiencia    |
| **Flow of Events**          | 1. El administrador accede a la sección de segmentación dentro del módulo de campaña.<br> 2. El sistema solicita que defina criterios de segmentación (edad, ubicación, intereses, etc.).<br> 3. El administrador selecciona los criterios adecuados para el público objetivo.<br> 4. El sistema guarda y aplica los criterios a la campaña en curso. |
| **Exceptions**              | - **Datos insuficientes**: Si no se completan todos los criterios necesarios, el sistema muestra un error y solicita completar la información. |
| **Preconditions**           | - El administrador debe tener acceso a los datos de usuarios segmentados.<br> - La campaña debe estar en fase de configuración. |
| **Postcondition**           | La campaña publicitaria se dirige al público segmentado según los criterios definidos. |
| **Includes Use Case**       | Definición de criterios de segmentación                                     |
| **Used Services**           | - Servicio de segmentación de audiencia <br> - Servicio de análisis de datos de usuarios |
| **Non-functional requirements** | - **Seguridad**: La información de los usuarios debe protegerse y cumplir con la normativa de privacidad.<br> - **Usabilidad**: El proceso de segmentación debe ser intuitivo y eficiente. |

### 5. Caso de uso: Programación de la Campaña

| **Realised User task**     | Programar la fecha y hora de inicio y fin de una campaña publicitaria. |
|----------------------------|------------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                             |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de programación de campañas |
| **Flow of Events**          | 1. El administrador accede a la sección de programación en el módulo de campañas.<br> 2. El sistema solicita ingresar la fecha y hora de inicio y finalización de la campaña.<br> 3. El administrador ingresa los datos y revisa el calendario de programación.<br> 4. El sistema guarda la programación y confirma la fecha seleccionada. |
| **Exceptions**              | - **Conflicto de programación**: Si hay campañas en curso que interfieren, el sistema alerta al administrador para evitar superposición. |
| **Preconditions**           | - El administrador debe tener los permisos para programar campañas.<br> - La campaña debe estar completamente configurada. |
| **Postcondition**           | La campaña publicitaria está programada para ejecutarse en las fechas especificadas. |
| **Includes Use Case**       | Calendario de campañas                                                 |
| **Used Services**           | - Servicio de calendario y programación <br> - Notificación de conflictos de campañas |
| **Non-functional requirements** | - **Disponibilidad**: El sistema debe estar disponible 24/7 para programar campañas.<br> - **Performance**: La programación debe completarse en menos de 2 segundos. |

---

### 6. Caso de uso: Ajuste de Campaña en Tiempo Real

| **Realised User task**     | Realizar ajustes a una campaña publicitaria activa. |
|----------------------------|----------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                         |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de campañas activas |
| **Flow of Events**          | 1. El administrador accede a la sección de campañas activas.<br> 2. El sistema muestra las campañas en curso con métricas en tiempo real.<br> 3. El administrador selecciona la campaña que desea modificar y ajusta parámetros como el presupuesto, la segmentación o los canales.<br> 4. El sistema aplica los cambios inmediatamente y actualiza la campaña. |
| **Exceptions**              | - **Error en la actualización**: Si el sistema no puede aplicar los cambios por problemas de conexión, muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - La campaña debe estar activa.<br> - El administrador debe tener los permisos necesarios para modificar campañas en curso. |
| **Postcondition**           | La campaña se ajusta según los nuevos parámetros definidos. |
| **Includes Use Case**       | Reconfiguración de campaña                                             |
| **Used Services**           | - Servicio de análisis en tiempo real <br> - Sistema de gestión de campañas activas |
| **Non-functional requirements** | - **Seguridad**: Los cambios deben realizarse solo por usuarios autorizados.<br> - **Performance**: Los cambios deben reflejarse en menos de 5 segundos. |

---

### 7. Caso de uso: Gestión de Presupuesto Publicitario

| **Realised User task**     | Asignar y gestionar el presupuesto para una campaña publicitaria. |
|----------------------------|-------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                        |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de gestión financiera |
| **Flow of Events**          | 1. El administrador accede a la sección de presupuesto en el módulo de campañas.<br> 2. El sistema solicita ingresar el monto del presupuesto para la campaña.<br> 3. El administrador define el monto y distribuye el presupuesto entre los diferentes canales publicitarios.<br> 4. El sistema guarda la configuración del presupuesto y permite monitorear los gastos en tiempo real. |
| **Exceptions**              | - **Presupuesto insuficiente**: Si el administrador intenta asignar un monto mayor al disponible, el sistema muestra un mensaje de error. |
| **Preconditions**           | - La campaña debe estar en fase de configuración o en curso.<br> - El sistema debe estar integrado con la gestión financiera de la empresa. |
| **Postcondition**           | El presupuesto ha sido asignado y está listo para ser utilizado en la campaña. |
| **Includes Use Case**       | Monitoreo de presupuesto                                          |
| **Used Services**           | - Servicio de gestión financiera <br> - Sistema de monitoreo de gastos publicitarios |
| **Non-functional requirements** | - **Seguridad**: El acceso a la gestión de presupuesto debe estar limitado a usuarios autorizados.<br> - **Performance**: Los ajustes al presupuesto deben aplicarse en menos de 3 segundos. |
