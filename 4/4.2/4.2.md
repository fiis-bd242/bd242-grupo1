# 4.2. Módulo de Campaña Publicitaria

### 1. Caso de uso: Creación de Campaña Publicitaria

| **Realised User task**     | Crear y configurar una campaña publicitaria en el sistema de Yape. |
|----------------------------|--------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                         |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de gestión de campañas |
| **Flow of Events**          | 1. El administrador accede al módulo de campañas publicitarias.<br> 2. El sistema solicita la configuración inicial de la campaña (nombre, objetivo, duración).<br> 3. El administrador ingresa los detalles solicitados y selecciona los canales donde se lanzará la campaña.<br> 4. El sistema guarda los datos y muestra una vista previa de la campaña.<br> 5. El administrador revisa y aprueba la configuración.<br> 6. El sistema programa la campaña para su lanzamiento en los canales seleccionados. |
| **Exceptions**              | - **Datos incompletos**: Si los campos obligatorios no están completos, el sistema muestra un mensaje de error y no permite avanzar hasta que se completen.<br> - **Problemas de conexión**: Si el sistema no puede conectarse a los canales publicitarios seleccionados, muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - El usuario debe tener permisos de administrador de marketing.<br> - El sistema debe estar conectado a las plataformas de publicidad. |
| **Postcondition**           | La campaña publicitaria ha sido creada y programada para su lanzamiento. |
| **Includes Use Case**       | Revisión de campaña, Aprobación de campaña                         |
| **Used Services**           | - Servicio de gestión de campañas publicitarias <br> - Servicio de conexión con canales publicitarios |
| **Non-functional requirements** | - **Usabilidad**: El proceso de creación de campañas debe ser intuitivo para el usuario.<br> - **Seguridad**: Solo los usuarios autorizados pueden crear campañas.<br> - **Performance**: La campaña debe guardarse y programarse en menos de 2 segundos. |

---

### 2. Caso de uso: Monitoreo de Campaña

| **Realised User task**     | Monitorear el desempeño de una campaña publicitaria activa. |
|----------------------------|-------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                  |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de análisis de campañas |
| **Flow of Events**          | 1. El administrador accede al módulo de monitoreo de campañas.<br> 2. El sistema muestra un tablero con las métricas clave de la campaña (impresiones, clics, conversiones).<br> 3. El administrador puede filtrar los datos por fecha, canal o segmentación.<br> 4. El sistema actualiza el tablero en tiempo real. |
| **Exceptions**              | - **Problemas de conexión**: Si hay problemas para obtener datos de los canales, el sistema muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - La campaña debe estar activa y lanzada en los canales seleccionados.<br> - El sistema debe tener acceso a las métricas de los canales publicitarios. |
| **Postcondition**           | El administrador obtiene un informe actualizado del rendimiento de la campaña. |
| **Includes Use Case**       | Generación de informes                                       |
| **Used Services**           | - Servicio de análisis de campañas <br> - Conexión con los canales publicitarios |
| **Non-functional requirements** | - **Performance**: Las métricas deben actualizarse en tiempo real.<br> - **Disponibilidad**: El sistema debe estar disponible para monitorear la campaña 24/7. |

---

### 3. Caso de uso: Generación de Informes Post-Campaña

| **Realised User task**     | Generar un informe detallado del rendimiento de la campaña publicitaria una vez finalizada. |
|----------------------------|--------------------------------------------------------------------------------------------|
| **Initiating Actors**       | Administrador de Marketing                                                                 |
| **Participating Actors**    | - Administrador de Marketing <br> - Sistema de generación de informes                      |
| **Flow of Events**          | 1. El administrador selecciona una campaña finalizada para generar el informe.<br> 2. El sistema extrae los datos de rendimiento (impresiones, conversiones, CTR, etc.).<br> 3. El sistema genera el informe en formato PDF o Excel.<br> 4. El administrador descarga el informe o lo envía a otras áreas. |
| **Exceptions**              | - **Error en la extracción de datos**: Si el sistema no puede obtener datos completos de la campaña, muestra un mensaje de error y permite reintentar. |
| **Preconditions**           | - La campaña debe haber finalizado.<br> - Los datos deben estar almacenados y accesibles.  |
| **Postcondition**           | El informe detallado del rendimiento de la campaña ha sido generado. |
| **Includes Use Case**       | Exportación de informes                                                                     |
| **Used Services**           | - Servicio de generación de informes <br> - Servicio de almacenamiento de datos de campañas |
| **Non-functional requirements** | - **Usabilidad**: El proceso de generación de informes debe ser rápido y sencillo.<br> - **Performance**: El informe debe generarse en menos de 5 segundos. |
